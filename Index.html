3

Why Q, K, V?

Query (Q): "What am I looking for?"  |  Key (K): "What do I offer?"  |  Value (V): "Actual content to gather".
Q and K compute attention scores, V provides the information to aggregate.

4

Computing Q, K, V (Matrix Multiplication)

Each embedding × learned weight matrices → Q, K, V vectors. Below: computed values for every token.

TokenQ (4-dim)K (4-dim)V (4-dim)
💡 Same operation, different weights — training learns W_Q, W_K, W_V.
5

The Key Insight

Q = W_Q × E , K = W_K × E , V = W_V × E. The "magic" is in learned weights. Matrix multiplication is fixed, but weights are tuned via backpropagation.

6

Attention Scores

For a query token, compute dot product with every token's Key, divided by √d (d=4 → √d=2).

7

Softmax → Attention Weights

Raw scores → probabilities (sum to 1). How much focus on each token.

8

Full Attention Matrix

Heatmap: rows = Query tokens, columns = Key tokens. Each row sums to 100%.

9

Output = Attention Weights × V

Contextualized embedding: weighted sum of Value vectors using attention probabilities.

10

The Complete Attention Flow

Embeddings → Q,K,V → Scores (Q·K/√d) → Softmax → Weighted sum of V → Contextual Embeddings.

⚡ [E] → Q,K,V → [Scores] → [Softmax] → [E'] = Attention(E)
11

From Attention to LLM

Real Transformers stack 96 layers, multi‑head attention, feed‑forward nets, residual connections & positional encoding. Attention is the core building block of GPT, BERT, etc.

12

Reasoning & Chain‑of‑Thought

LLMs predict next token. Chain‑of‑Thought (CoT) breaks reasoning into steps, each step becomes context for the next prediction — easier than one giant leap.

📌 Final formula: Attention(Q,K,V) = softmax(Q·Kᵀ / √d) · V
